機能

  • 接続について
    • 多段接続

      gateway を通して SFTP サーバに接続する事が可能です。 つまり、例えば ssh server1 -> ssh server2 -> ssh sftpserver として入る事の出来る sftpserver の内容にアクセスする事ができます。 接続の段数は何段でも増やせます (但し、当然ですがその分動作は遅くなります)。

    • 自動接続・切断

      使用していない時は自動で接続を切断します。 最後に使用してから切断に到るまでの時間を指定する事が可能です。 また、切断時に使用しようとすると自動的に再接続をします。

    • Heartbeat 送信

      長時間使用していないが接続状態を維持したい時、 接続を保持する為に空のメッセージをサーバに送信します。 長時間通信がないと NAT Entry が失われる場合に利用できます。 Heartbeat の間隔を指定する事が可能です。

    • エラー時の再接続

      接続のエラーによって操作に失敗した時、 自動的に再接続をして再度同じ操作を試行します。 一つの操作に対する再接続の最大試行回数を指定する事が可能です。

  • キャッシュ機能

    動作を高速にする為にキャッシュ機能を備えています。

    • 書込プロテクト

      内容を閲覧するだけで変更操作を実行したくないサーバについては、 読込専用の設定を適用する事ができます。

    • ファイル情報のキャッシュ

      取得したファイル情報をキャッシュする設定を選択できます。 最新の情報が手許にある場合には、キャッシュされた情報を使用します。 手許の情報が古くなっている場合には、再度取得を試みます。 (現在は 5 秒で情報の期限が切れる設定にしています。)

    • ファイル内容の読込キャッシュ

      ファイル内容の読込をキャッシュする設定を選択できます。 既に読み込んだ内容はローカルにキャッシュします。 リモートへの要求は 4KiB のブロック単位で行われ、 小さい要求が大量に実行されるのを防ぎます。 また、シーケンシャルにアクセスしている場合には先読みを実行します。 (リモートでファイル内容が更新された場合は、 ファイル時刻でそれを検出しキャッシュを破棄します。)

    • 変更操作のキャッシュ

      ファイルの移動、削除、ファイルへの書込などの操作をキャッシュする設定を選択できます。 大量のデータを書き込んだり大量のファイルを移動した場合などはリモートの更新に時間が掛かりますが、 この設定を有効にしておくと手許では恰も操作がすぐに完了した様に見せて、 バックグラウンドで時間をかけて更新をリモートに適用します。

  • ファイルシステム
    • 複数の接続先

      複数の接続先を指定する事ができます。

    • シンボリックリンク

      リモートのシンボリックリンクの表示の方法を選択する事ができます: i) 恰もファイル・ディレクトリ本体が其処にあるかの様に表示します。 ii) 或いは、Cygwin の symlink ファイルとして表示します。

    • ファイル名の特殊文字

      * や <, > や ? など特別な文字を含んだファイル名が Unix 系 OS で使われている可能性があります。 その様なファイルに対しても、特別な文字を Unicode 私用領域の文字に対応させるので、適切に操作を行う事が可能です。 この動作は Cygwin に倣った物です (Cygwin アプリケーションからは特別な文字のファイル名として操作ができます!)。

    • ZoneID (改良予定)
        セキュリティの為に ZoneID を設定しています。 (■予定: ZoneID を設定画面で変更できる様にする予定です)
